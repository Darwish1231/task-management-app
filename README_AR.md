# TaskHub - نظام إدارة مهام ذكي وسياقي

TaskHub هو تطبيق متكامل لإدارة المهام، مصمم لتعزيز إنتاجيتك من خلال الربط الذكي مع GitHub وتقديم رؤى برمجية ومعمارية من Hacker News.

## 🚀 نظرة عامة على المشروع

معظم تطبيقات إدارة المهام تعمل بشكل منعزل، لكن TaskHub يربط قائمة مهامك بالنظام البرمجي الخارجي الخاص بك.
- **مزامنة GitHub**: يمكنك تعديل حالة المهمة وأولويتها تلقائياً بمجرد ربطها بمسار (Issue) على GitHub.
- **رؤى سياقية**: احصل على أخبار ومقالات برمجية معمارية مرتبطة بموضوع المهمة التي تعمل عليها حالياً.
- **واجهة مستخدم عصرية (Glassmorphism)**: تصميم متميز ومتجاوب يركز على الإنتاجية والجماليات.

---

## 🛠 التقنيات المستخدمة (Tech Stack)

### الجزء الخلفي (Backend)
- **Node.js & Express**: إطار عمل سريع لخدمات الويب.
- **MongoDB & Mongoose**: قاعدة بيانات مرنة تعتمد على المستندات (NoSQL).
- **JWT (JSON Web Tokens)**: نظام مصادقة آمن وعديم الحالة.
- **Express-Validator**: نظام متطور للتحقق من صحة البيانات (Validation).
- **Axios**: عميل HTTP للتعامل مع الواجهات البرمجية الخارجية.

### الجزء الأمامي (Frontend)
- **React (Vite)**: مكتبة واجهة مستخدم حديثة مع أداة بناء فائقة السرعة.
- **React Context API**: لإدارة الحالة المركزية للمصادقة (Auth).
- **Vanilla CSS**: تصميم مخصص باستخدام متغيرات CSS لإنشاء نظام تصميم فريد.
- **Lucide React**: أيقونات نظيفة ومتناسقة.

## 📂 هيكل المجلدات (Folder Structure)

تم تصميم المشروع باستخدام هيكل "منظم بالخدمات" (Service-Oriented) لضمان فصل المهام (Separation of Concerns):

### الجزء الخلفي (`/backend`)
- `src/controllers/`: مسؤول عن التعامل مع الطلبات (Requests) وتنسيق الاستجابة.
- `src/services/`: يحتوي على منطق العمل الأساسي والتعامل مع الـ APIs الخارجية.
- `src/validators/`: يحدد المخططات الصارمة للتحقق من صحة البيانات المدخلة.
- `src/models/`: ملفات تعريف قاعدة البيانات (Schemas).
- `src/middleware/`: يتعامل مع معالجة الأخطاء والأمان (Auth).
- `src/utils/`: أدوات مشتركة مثل نظام الذاكرة المؤقتة (Memory Cache).

### الجزء الأمامي (`/frontend`)
- `src/pages/`: الشاشات الأساسية (لوحة التحكم، تسجيل الدخول، إنشاء حساب).
- `src/components/`: عناصر واجهة المستخدم القابلة لإعادة الاستخدام.
- `src/context/`: مزود الحالة العامة للمصادقة.
- `src/services/`: إعدادات الربط مع الـ Backend.

## ⚙️ كيفية التشغيل (How to Run)

### المتطلبات الأساسية
- Node.js (الإصدار 18 فما فوق)
- MongoDB (محلي أو عبر Atlas Cluster)

### 1. إعداد الـ Backend
```bash
cd backend
npm install
```
قم بإنشاء ملف `.env` في مجلد `backend`:
```env
PORT=5001
MONGO_URI=رابط_قاعدة_بيانات_مونجو
JWT_SECRET=كلمة_السر_الخاصة_بك
```
تشغيل السيرفر:
```bash
npm run dev
```

### 2. إعداد الـ Frontend
```bash
cd frontend
npm install
```
تشغيل الجزء الأمامي:
```bash
npm run dev
```

## 🌐 الواجهات البرمجية الخارجية (External APIs)

1. **GitHub API**: يستخدم لجلب بيانات الـ Issue (الحالة، التصنيفات، العنوان).
   - *لماذا*: للسماح للمبرمجين بربط مهامهم بمشاكل البرمجة الفعلية، وتحديث الحالة تلقائياً عند إغلاق الـ Issue على GitHub.
2. **Algolia HackerNews API**: يستخدم لجلب مقالات معمارية ذات صلة.
   - *لماذا*: لتقديم "إثراء سياقي". إذا قمت بإنشاء مهمة عن "تحسين قواعد البيانات"، سيقترح عليك التطبيق قراءة مقالات حديثة حول هذا الموضوع.

## ⚠️ القيود الحالية (Known Limitations)
- **الذاكرة المؤقتة (Memory Cache)**: يستخدم السيرفر ذاكرة مؤقتة بسيطة، يتم مسحها عند إعادة تشغيل السيرفر.
- **المصادقة عديمة الحالة**: يعتمد على JWT الأساسي. في بيئة الإنتاج، يفضل استخدام Refresh Tokens.
- **تحتاج لاتصال دائم**: التطبيق يحتاج لاتصال بالإنترنت لجلب البيانات من GitHub و HackerNews.
